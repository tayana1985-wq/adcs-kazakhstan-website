# ADCS KZ Website

Статический сайт ADCS Kazakhstan: национальная платформа корпоративного спорта Казахстана.

## Локальный запуск

```bash
python3 -m http.server 5173
```

После запуска откройте:

```text
http://127.0.0.1:5173/index.html
```

## Проверка / сборка

Сайт не требует сборки: это статические HTML, CSS и JS-файлы.

Для технической проверки структуры:

```bash
npm run build
```

Если `npm` локально не установлен, можно запустить тот же проверочный скрипт напрямую:

```bash
node scripts/validate-site.mjs
```

Команда проверяет, что:

- все локальные HTML-ссылки ведут на существующие страницы;
- все языковые версии страниц существуют;
- общее меню содержит 11 пунктов;
- для `ru`, `kk`, `en` есть переводы всех пунктов меню;
- все страницы подключают `data.js` и `site.js`.

## Публикация на Vercel

1. Загрузите папку проекта в GitHub.
2. Откройте Vercel и выберите `Add New Project`.
3. Подключите GitHub-репозиторий с этим сайтом.
4. Framework Preset: `Other`.
5. Build Command: `npm run build`.
6. Output Directory: оставьте пустым или укажите корень проекта.
7. Нажмите `Deploy`.

Файл `vercel.json` уже добавлен. Clean URLs отключены, чтобы статические `.html`-маршруты не ломали относительные подключения CSS, JS и assets.

## Публикация на GitHub Pages

1. Создайте репозиторий на GitHub.
2. Загрузите содержимое этой папки в ветку `main`.
3. В настройках репозитория откройте `Settings` -> `Pages`.
4. Source: `Deploy from a branch`.
5. Branch: `main`, folder: `/root`.
6. Сохраните настройки и дождитесь публикации.

Для GitHub Pages добавлен файл `.nojekyll`, чтобы GitHub не обрабатывал сайт через Jekyll.

## Публикация через Netlify Drop

1. Откройте Netlify Drop.
2. Перетащите всю папку `CorporateSportWebsite` в окно загрузки.
3. Build-команда не нужна.
4. Publish directory не нужен, потому что `index.html` находится в корне папки.

Все локальные подключения используют относительные пути `./style.css`, `./data.js`, `./site.js`, `./assets/...`.

## Где редактировать сайт

- Меню и маршруты: `data.js`, блоки `menuItems`, `routes`, `translations`.
- Языки и названия пунктов меню: `data.js`, блок `translations`.
- Виды спорта: `data.js`, блоки `sportCategories` и `sports`.
- События и календарь: `data.js`, блок `events`.
- Партнёрские пакеты: `data.js`, блок `partnerPackages`.
- Ценовые пакеты: `data.js`, блок `pricingPackages`.
- Команда: `data.js`, блок `teamMembers`.
- Контакты: `data.js`, блок `contacts`.
- Внешний вид: `style.css`.
- Логика меню, языкового переключателя, карточек и форм: `site.js`.

## Мультиязычность

Меню строится из одного общего массива `menuItems`. Для каждого языка берутся только переводы подписей:

- `ru`: русские названия;
- `kk`: казахские названия;
- `en`: английские названия.

Если перевод контента страницы пока не готов, страница всё равно существует и показывает русский fallback-контент. Страницы не скрываются из меню из-за отсутствия перевода.
